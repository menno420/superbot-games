## 💡 Session idea

The `upgrade --apply-docs` post-hoc path re-runs after the main upgrade and
rewrites `.substrate/upgrade-report.md`, but the rewritten report no longer
distinguishes "applied this run" from "was already applied" — a later agent
reading the report can't tell whether the template-improved doc was taken
or is still pending without diffing git history. Kit-side candidate: the
report's per-doc `note` column should flip to `applied YYYY-MM-DD via
--apply-docs` once taken, so the report stays a truthful single-glance
ledger of what landed versus what is still on offer.
