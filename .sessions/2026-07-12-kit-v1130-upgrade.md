## 💡 Session idea

`bootstrap.py upgrade` should end by running its own `check --strict` and
printing any NON-designed findings the upgrade itself just introduced. This
wave, replanting `docs/SKILLS.md` created an orphan `[reachable]` red that
only surfaced at the verify step — an upgrade that can introduce a gate-
blocking finding should say so in its own output ("upgrade introduced 1
finding: SKILLS.md orphan — wire it via the AGENT_ORIENTATION delta below")
so distribution sessions see the fix path immediately instead of
discovering it mid-verify.
